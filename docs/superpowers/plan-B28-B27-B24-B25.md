**Step 2 — Refactor `_compute_regime_series()` to delegate to `eval_rules()` when `rules` present (backend)**

File: `backend/routes/backtest.py`, function `_compute_regime_series()` (line 215).

Logic:
```
if rc.rules:
    # Build HTF OHLCV + compute all indicators needed by rc.rules
    # Call eval_rules() per HTF bar → raw_bool boolean Series
else:
    # Existing single-indicator path (unchanged)
```

For the new `rules` path:
1. Compute `lookback` as `max(htf_lookback_days(r.indicator, r.params or {}) for r in rc.rules)` — use the largest warm-up needed across all rules in the regime rule set. Import `htf_lookback_days` from `shared` (already imported).
2. Fetch HTF data with the extended start (same as existing code).
3. Build `ohlcv` from HTF df (same pattern as existing).
4. Call `compute_indicators(htf_close, high=htf_high, low=htf_low, volume=htf_vol, rules=migrated_rules)` — this returns the indicator dict for the HTF timeframe.
5. Iterate over the HTF index, calling `eval_rules(migrated_rules, rc.logic, htf_indicators, i)` → produce a boolean Series indexed on HTF bars.
6. Apply `min_bars` smoothing (rolling min, same as existing lines 260–263).
7. `align_htf_to_ltf()` and return (same as existing lines 265–266).

Migrate rules before evaluation: `migrated_rules = [migrate_rule(r) for r in rc.rules]`. Import `migrate_rule` — already imported on line 9.

Why `compute_indicators` with HTF OHLCV: `compute_indicators` builds indicator series for whatever rules are passed. The HTF `close`/`high`/`low` are the correct inputs for a daily/weekly regime.

Dependency: Step 1.

**Step 3 — Refactor `_eval_regime_direction()` in `regime.py` to use `eval_rules()` when `rules` present (backend)**

File: `backend/regime.py`, method `_eval_regime_direction()` (lines 23–87).

**Step 2 — Implement `importFromStrategy()` handler (frontend)**

File: `frontend/src/features/strategy/StrategyBuilder.tsx`

```typescript
function importFromStrategy(tab: 'regime' | 'long' | 'short', sourceName: string) {
  const source = savedStrategies.find(s => s.name === sourceName)
  if (!source) return

  // Prefer direction-specific rules when available, fall back to single-mode rules
  let sourceBuy: Rule[]
  let sourceSell: Rule[]
  if (tab === 'long') {
    sourceBuy = source.longBuyRules?.length ? source.longBuyRules : source.buyRules
    sourceSell = source.longSellRules?.length ? source.longSellRules : source.sellRules
  } else if (tab === 'short') {
    sourceBuy = source.shortBuyRules?.length ? source.shortBuyRules : source.buyRules
    sourceSell = source.shortSellRules?.length ? source.shortSellRules : source.sellRules
  } else {
    // Regime tab: always use single-mode buyRules (regime rules are always single-mode)
    sourceBuy = source.buyRules
    sourceSell = source.sellRules
  }

  const migratedBuy = sourceBuy.map(migrateRule)
  const migratedSell = sourceSell.map(migrateRule)

  // Guard: if source has no entry rules, it was likely saved in regime mode. Warn and abort.
  if (migratedBuy.length === 0) {
    // Show inline warning (e.g. set a `importError` state string, render near the select)
    setImportError(`"${sourceName}" has no entry rules — it may have been saved in regime mode. Try importing its sub-strategies instead.`)
    setImportingTab(null)
    return
  }

  // Confirm if destination already has rules
  const destCount = tab === 'regime' ? regimeBuyRules.length : tab === 'long' ? longBuyRules.length : shortBuyRules.length
  if (destCount > 0 && !window.confirm(`Replace ${destCount} existing rule${destCount > 1 ? 's' : ''} with rules from "${sourceName}"?`)) {
    setImportingTab(null)
    return
  }

  if (tab === 'regime') {
    setRegimeBuyRules(migratedBuy)
    setRegimeLogic(source.buyLogic ?? 'AND')
  } else if (tab === 'long') {
    setLongBuyRules(migratedBuy)
    setLongSellRules(migratedSell)
    setLongBuyLogic(source.buyLogic ?? 'AND')
    setLongSellLogic(source.sellLogic ?? 'AND')
  } else if (tab === 'short') {
    setShortBuyRules(migratedBuy)
    setShortSellRules(migratedSell)
    setShortBuyLogic(source.buyLogic ?? 'AND')
    setShortSellLogic(source.sellLogic ?? 'AND')
```

```python
    """Return per-direction stop_loss_pct when b23_mode, else global."""

def _dir_ts(direction: str) -> Optional[TrailingStopConfig]:
    """Return per-direction trailing_stop when b23_mode, else global.
    Merges per-direction type+value into a copy of the global config's other fields
    (source, activate_on_profit, activate_pct) so those settings are never lost.
    """

        return base

        return override

        type=override.type, value=override.value,
        source=base.source, activate_on_profit=base.activate_on_profit,
        activate_pct=base.activate_pct
    )

def _dir_mbh(direction: str) -> Optional[int]:
    """Return per-direction max_bars_held when b23_mode, else global."""
    if not b23_mode:

    return req.short_max_bars_held if req.short_max_bars_held is not None else req.max_bars_held
```
