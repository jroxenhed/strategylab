## Friction points

- **`backend/venv/` still missing** — 6th consecutive build hitting this. `pip install --quiet` from the routine PATH worked (root warning aside), but every build wastes the `pip install` round-trip on the same set of deps. Container image needs `backend/venv/` baked in. F97 still unaddressed.
- **Main was force-updated overnight** — the routine builder's local `main` is stale from the previous session's checkout. `git pull --ff-only` aborts because the remote diverged. Needed to `git reset --hard origin/main`, which the protocol doesn't explicitly authorize but is the only safe move here (the builder branch's commits land in PRs that get merged via squash+force-push, which rewrites main's history). Worth codifying in pre-flight #2.
- **TODO body inaccuracy on F145** — the body said "BatchQuickBacktestRequest.symbols is already `list[SymbolField]` (F91)" but the actual code was `list[str]` + mode='before' validator that called `normalize_symbol` per element. Caught early via Read. The body might have been written before F91 actually shipped, or written aspirationally. Suggests TODO-body accuracy decays when items sit unimplemented for 1-2 builds.
- **F65 already-shipped trap** — F65 was filed by an architectural audit on 2026-05-10, F135 shipped the fix on 2026-05-11, but nobody linked them. Cost a `grep` and a Read to confirm, then a "closed as subsumed" TODO update. Mildly frustrating but not actually broken — the audit-vs-implementation linkage just needs a manual reconciliation pass each session, which this build did.

## Recommendations

1. **Codify the force-update branch handling in pre-flight #2.** Add to `docs/overnight-builder-prompt-patch.md` §Pre-flight #2: "If `git pull --ff-only` aborts because main diverged from local, `git reset --hard origin/main` is authorized — squash+force-push on PR merge rewrites main's history, and the builder's local branch was based on the pre-rewrite main. Note in NEXT_RUN.md."
2. **F97 escalation.** 6 consecutive builds with `backend/venv/` missing. The pip-install workaround is functional but wasteful. Filing as recommendation only — actual container change is out of builder scope.
3. **TODO body accuracy decay.** Suggest a quick pre-implementation grep against the body's "already X" / "still Y" claims, especially for items filed >2 builds ago. F145's body was 1 build out of date; harmless here but could mislead in higher-stakes refactors. Not worth automating — just a builder-discipline note.
4. **Add a "closed as subsumed" pattern to the TODO sync script.** F65 was closed by F135 a build ago but stayed unchecked. The sync script could greppingly flag items where an unchecked F-item's body references a different already-shipped F-id, surfacing candidate subsumption pairs for manual review. Filed as note only — not file-able as a TODO item without more design.
